📊 Field Service Report Automation System

📖 Overview

Jehovah’s Witnesses submit a monthly field service report to congregation elders. Traditionally, this process is done either:
	•	On paper forms
	•	Via SMS or messaging apps sent to the group overseer

The overseer then manually records and stores these reports.

This project aims to digitize and automate that process using a web-based system.

⸻

🚀 Problem

The current system has several limitations:
	•	❌ Manual record keeping is time-consuming
	•	❌ Reports can be lost or forgotten
	•	❌ No centralized or easily searchable database
	•	❌ No automatic reminders for publishers

⸻

💡 Solution

This web application provides a simple and efficient way to submit and manage field service reports.

Key Features:
	•	📱 SMS Reminder System
	•	Automatically sends reminders at the end of each month
	•	Includes a link to the web app for quick submission
	•	🌐 Web-Based Report Submission
	•	Users can submit reports through a clean interface
	•	Accessible from any device
	•	🗂 Centralized Database
	•	All reports are stored securely
	•	Easy retrieval and tracking
	•	👥 User & Group Management
	•	Organize publishers into groups
	•	Assign overseers to groups

⸻

🛠 Tech Stack
	•	Backend: FastAPI
	•	Database: PostgreSQL
	•	ORM: SQLAlchemy
	•	Language: Python

⸻

⚙️ How It Works
	1.	At the end of the month, users receive an SMS reminder
	2.	The SMS contains a link to the web application
	3.	Users submit their field service report online
	4.	Data is stored in the database and accessible to overseers
